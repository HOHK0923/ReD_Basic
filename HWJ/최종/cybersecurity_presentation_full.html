침투 테스터의 **공격 자동화 속도**는 점점 빨라지는 반면, 기업의 **수동적인 탐지 및 대응 시간(MTTR)**은 여전히 느려, 방어 체계가 실제 위협에 뒤처지고 있습니다.

해결 방안: 대응의 자동화

Red Team의 **공격 자동화**를 통해 취약점을 신속하게 발굴하고, Blue Team의 **실시간 대응 자동화(SOAR)**를 통해 MTTR을 획기적으로 줄이는 통합 검증 환경 구축.

Layer 2 Detail: 위협 점수화 시스템 (Threat Scoring)

단일 공격 vs 복합 공격 판단

단순 오탐이 아닌 실제 공격을 식별하기 위해 공격 유형별 **가중치**를 부여하고, **N분 동안 누적 점수**가 임계치(Threshold)를 초과할 경우 High-Risk로 분류하는 시스템을 SPL로 구현했습니다.

공격 유형 (Rule ID) 가중치 (Weight) 임계치 (Threshold)
SQLi / SSRF (Critical) +10점 **20점 초과 시**
(Fail2Ban 자동 차단 트리거)
XSS / CSRF (High) +5점
WAF Bypass Attempt (Medium) +1점

Layer 3 Detail: Fail2Ban Custom Filter

정확도 높은 `failregex` 개발

**CRS Rule ID와 Custom Rule ID(999xxx)**를 기반으로 하는 정규 표현식(`failregex`)을 작성하여 오탐을 최소화하고 Red Team의 High-Risk 공격만을 정확하게 차단합니다.

failregex = [modsecurity] ... "id:(942100|941100|930100|999001)"

# jail.local (Jail)
[modsec-waf]
enabled = true
filter = modsec_high_threat
maxretry = 3
